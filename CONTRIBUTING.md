# Contributing to Board Taxonomies

Thank you for your interest in contributing to the Board Taxonomies project! This document provides guidelines for contributing to this taxonomy.

## How to Contribute

### Reporting Issues

If you find any of the following, please open an issue:

- Incorrect or unclear category definitions
- Boards that are miscategorised
- Missing board categories
- Typos or formatting issues
- Broken links

### Suggesting Changes

We welcome suggestions for:

1. **New board categories** - If you believe there's a distinct category not covered
2. **Additional examples** - More board examples for existing categories
3. **Clarifications** - Improvements to existing definitions
4. **Edge cases** - Boards that don't fit well into current categories
5. **Form factor updates** - New or emerging form factors

### Submitting Pull Requests

1. **Fork the repository** and create a new branch for your changes
2. **Make your changes** - Be clear and concise in your modifications
3. **Test your changes** - Ensure all links work and formatting is correct
4. **Write a clear commit message** - Describe what you changed and why
5. **Submit a pull request** - Reference any related issues

#### Pull Request Guidelines

- One logical change per pull request
- Include reasoning for categorisation changes
- Provide sources/references when possible
- Ensure markdown formatting is consistent
- Update the Table of Contents if you add new sections

### Adding New Board Examples

When adding a new board example, please ensure:

- The board clearly fits the category definition
- It's a real, commercially available or well-known board
- You provide enough context if it's lesser-known
- It doesn't duplicate an existing example

### Proposing New Categories

If proposing a new category, please include:

1. **Clear definition** - What distinguishes this category?
2. **Key characteristics** - Specific technical attributes
3. **Multiple examples** - At least 3-5 example boards
4. **Justification** - Why existing categories don't cover this
5. **Common form factors** - If applicable

### Classification Disputes

If you disagree with how a board is categorised:

1. Open an issue explaining your reasoning
2. Reference the specific characteristics from the definitions
3. Suggest an alternative classification
4. Be open to discussion - many boards blur category lines

## Style Guidelines

- Use clear, technical language
- Keep explanations concise but complete
- Maintain consistent formatting with existing content
- Use bulleted lists for characteristics
- Include both common and technical terms where applicable

## Questions?

If you're unsure about anything, feel free to:

- Open an issue to ask questions
- Start a discussion in pull requests
- Reference similar examples in the existing taxonomy

## Code of Conduct

Please note that this project has a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to abide by its terms.

## Recognition

All contributors will be recognised in the project. Significant contributions may be highlighted in release notes or changelog updates.

Thank you for helping improve this taxonomy!
